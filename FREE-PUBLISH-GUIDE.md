## 🥇 第一梯队：零成本、大流量（强烈推荐）

### 1. Google Play — $25 终身（几乎免费）
| 项目 | 说明 |
|------|------|
| 注册费 | **$25 一次性**（终身有效，不再收其他费用） |
| 审核时间 | 1~3 天 |
| 用户量 | 全球最大安卓市场 |
| 个人开发者 | ✅ 支持 |

**为什么说"几乎免费"**：$25 是一次性费用，之后永远不再交钱。对比其他渠道的隐性成本，这是性价比最高的选择。

**操作步骤**：
1. 打开 https://play.google.com/console → 用 Google 账号登录
2. 支付 $25 注册费（支持信用卡/Visa）
3. 创建应用 → 填写 STORE-LISTING.md 中的素材
4. 上传 AAB 包（用 PWABuilder 生成）
5. 提交审核

### 2. GitHub Releases — 100% 免费
| 项目 | 说明 |
|------|------|
| 费用 | **完全免费** |
| 流量 | 全球开发者社区 + SEO 收录 |
| 操作难度 | ⭐ 极简 |
| 用户获取 | 通过 GitHub 搜索、分享链接发现 |

**操作步骤**：
1. 在你的仓库 [releases](https://github.com/drgonw1021-lgtm/ringlight/releases) 页面
2. 点击 "Create a new release"
3. 上传 APK 文件作为 release asset
4. 写好 Release Notes（更新日志）
5. 发布！用户可以直接下载安装

**优点**：
- 零成本、零审核、即时发布
- 版本管理清晰（每个 release 一个版本）
- 自动生成 QR 码方便手机扫码下载

**如何生成 APK**：
```
方法 A：PWABuilder 打包时选择 Android → 下载 zip 解压得 APK/AAB
方法 B：Bubblewrap CLI 打包（需要 JDK）
```

## ⚠️ 注意事项

1. **签名一致性**：所有渠道的 APK 必须用同一个 keystore 签名，否则用户无法升级
2. **版本号递增**：每次发新版本必须增加 versionCode（1, 2, 3...），versionName 对应显示名（1.0.0, 1.1.0...）
3. **截图要真实**：Google Play 会检测虚假截图，用真机截取
4. **隐私政策是硬性要求**：Google Play / Amazon 必填，已为你准备好
